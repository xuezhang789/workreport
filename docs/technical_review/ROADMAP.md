# 分步优化实施方案 (Roadmap)

## 第一阶段：基石修复与安全加固 (Week 1-2)
**目标**: 消除高危风险，确保系统稳定运行。

*   **Week 1: 安全修复**
    *   [x] 禁用开放注册，完善账号体系。
    *   [x] 修复 `username_check_api` 枚举漏洞。
    *   [ ] 全局 CSRF 策略审查与加固。
    *   [ ] 敏感配置 (SECRET_KEY, DB密码) 环境变量化检查。
*   **Week 2: 关键 Bug 修复**
    *   [x] 修复任务导出 N+1 性能问题。
    *   [x] 优化效能看板计算逻辑 (DB Aggregation)。
    *   [ ] 修复已知的 UI 交互小问题 (如移动端表格溢出)。
    *   [ ] 建立 Sentry 错误监控。

## 第二阶段：核心流程优化 (Week 3-5)
**目标**: 提升核心业务（任务、日报）的操作效率和性能。

*   **Week 3: 性能攻坚**
    *   [ ] 引入 Redis 缓存层，缓存权限计算和统计结果。
    *   [ ] 数据库索引深度优化 (根据慢查询日志)。
    *   [ ] 日报自动保存 (Auto-save) 功能开发。
*   **Week 4: 体验升级**
    *   [ ] 任务详情页改版为“抽屉式”交互。
    *   [ ] 引入 Command Palette (Cmd+K) 快捷操作。
    *   [ ] 优化富文本编辑器体验 (支持粘贴图片上传)。
*   **Week 5: 自动化与通知**
    *   [ ] 集成飞书/钉钉机器人通知。
    *   [ ] 优化邮件通知模板，支持聚合发送（避免邮件轰炸）。

## 第三阶段：功能扩展与智能化 (Week 6-9)
**目标**: 引入新特性，提升管理价值。

*   **Week 6-7: 移动端适配**
    *   [ ] 完善 RESTful API 接口覆盖。
    *   [ ] 开发 H5 轻量版，嵌入钉钉/飞书工作台。
*   **Week 8: 高级报表**
    *   [ ] 开发任务甘特图视图。
    *   [ ] 实现自定义报表生成器 (拖拽字段生成报表)。
*   **Week 9: AI 探索**
    *   [ ] 接入 LLM API，实现日报周报自动摘要。
    *   [ ] AI 助手：自然语言搜索任务 ("查找上周张三未完成的Bug")。

## 第四阶段：架构升级 (Week 10+)
**目标**: 支撑更大规模用户量。

*   **Week 10**: 数据归档机制 (历史数据冷热分离)。
*   **Week 11**: 微服务化评估 (是否拆分 Report 独立服务)。
*   **Week 12**: 容器化与 K8s 部署方案标准化。

## 验收标准
1.  **安全**: 通过自动化安全扫描 (OWASP ZAP)，无高危漏洞。
2.  **性能**: 核心页面加载速度 < 1s (P95)。
3.  **质量**: 核心业务单元测试覆盖率 > 80%。
