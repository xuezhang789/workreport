# 人事管理模块 (Personnel Management) 审查与优化报告

**日期**: 2026-02-28
**审查对象**: 人事管理模块 (HR Module)
**审查范围**: 视图逻辑 (`views_hr.py`), 前端模板 (`personnel_list.html`), 数据模型 (`Profile`)

---

## 1. 现状评估 (Current Status)

人事管理模块目前提供了一个基础的员工档案管理界面，仅超级管理员 (`superuser`) 可访问。

### 1.1 核心功能
*   **列表展示**: 展示员工基本信息、职位、状态、入职日期、薪资及备注。
*   **档案编辑**: 通过模态框 (Modal) 编辑员工状态、日期、薪资及中介信息。
*   **统计分析**: 顶部展示总人数、在职/离职人数及本月新入职人数。
*   **数据导出**: 支持 CSV 格式导出筛选后的数据。

### 1.2 数据模型 (Profile)
模型设计覆盖了核心人事字段：
*   **基础**: `position`, `employment_status`, `hire_date`, `resignation_date`
*   **薪资**: `probation_salary`, `official_salary`, `currency`
*   **中介**: `intermediary_company`, `fee`, `currency`

---

## 2. 关键问题与风险 (Key Issues & Risks)

### 2.1 安全隐患 (Security)
*   **薪资信息裸露**: 在列表页直接展示所有员工的试用期和正式薪资。如果管理员在公共场合（如开放式办公室或会议投屏）访问此页面，极易导致**严重的薪资泄露事故**。
*   **敏感操作审计**: 虽然有 `log_action`，但缺乏专门的薪资变更历史记录表，难以追踪具体的调薪幅度和时间点。

### 2.2 功能缺失 (Missing Features)
*   **缺乏薪资历史 (Salary History)**: 系统仅存储“当前薪资”。HR 无法查看员工过去的调薪记录，无法进行薪资趋势分析。
*   **缺乏合同管理**: 没有字段记录劳动合同的开始/结束日期，也不支持上传合同 PDF 附件。
*   **新增员工入口**: 目前依赖用户自行注册或从其他模块导入，HR 无法直接在后台“预录入”新员工信息。

### 2.3 用户体验 (UX/UI)
*   **操作繁琐**: “离职”操作需要点击编辑 -> 修改状态 -> 选择日期 -> 保存。缺乏“一键离职”快捷方式。
*   **信息过载**: 表格列数较多，在小屏幕上体验不佳。

---

## 3. 优化建议 (Optimization Recommendations)

### 3.1 安全性增强 (Security)
*   **薪资脱敏显示**: 
    *   **方案**: 在列表页默认将薪资显示为 `******` 或 `Click to view`。点击后通过 JS 切换显示（甚至可以要求二次确认）。
    *   **实施**: 修改 `personnel_list.html`，使用 CSS 类控制显隐。

### 3.2 功能扩展 (Feature Enhancement)
*   **薪资变更历史**:
    *   **建议**: 新建 `SalaryHistory` 模型 (`user`, `old_salary`, `new_salary`, `change_date`, `reason`, `operator`)。
    *   **逻辑**: 每次修改薪资时，自动触发创建一条历史记录。
*   **合同附件管理**:
    *   **建议**: 在 `Profile` 或新建 `Contract` 模型中增加 `FileField`，允许上传 PDF。

### 3.3 交互体验优化 (UX)
*   **快捷操作菜单**: 在“操作”列增加下拉菜单 (Dropdown)，提供：
    *   `转正` (自动设置状态为 Active，更新薪资)
    *   `离职办理` (弹出专用模态框，仅需填离职日期和原因)
*   **智能搜索**: 优化搜索框，支持回车即搜，并增加防抖 (Debounce) 机制。

### 3.4 逻辑修正
*   **薪资校验规则**: 后端 `update_hr_info` 强制要求“正式薪资 > 试用薪资”。建议改为**警告**而非**错误**，因为存在平薪转正或特殊调岗情况。

---

## 4. 实施路线图 (Implementation Roadmap)

### Phase 1: 安全与体验修复 (立即执行)
1.  **UI**: 修改列表页，默认隐藏薪资数字，增加“显示/隐藏”切换按钮。
2.  **UX**: 优化搜索交互，支持回车搜索。
3.  **Fix**: 放宽后端薪资校验逻辑 (允许 Official == Probation)。

### Phase 2: 核心功能补全 (本周内)
1.  **Feature**: 增加“一键离职”和“一键转正”快捷操作。
2.  **Model**: 创建 `SalaryHistory` 模型，并迁移现有变更日志逻辑。

### Phase 3: 高级特性 (下个迭代)
1.  **Feature**: 合同管理与附件上传。
2.  **Feature**: 薪资分析图表（部门薪资分布、调薪趋势）。

---

## 5. 验收标准 (Acceptance Criteria)
*   [ ] 列表页打开时，薪资字段默认不可见。
*   [ ] 管理员可以查看并操作所有员工的档案。
*   [ ] 修改薪资后，能在（未来实现的）历史记录中查到变更详情。
*   [ ] 即使正式薪资等于试用薪资，也能保存成功。
